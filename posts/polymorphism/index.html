Polymorphism and data type

多态∣   java∣   2020-02-25  1194字

想起好久没去看过王垠的博客，今晚上去发现了他新发布了一篇博文，问了一个关于java类型的问题，我觉得十分有趣，刚好我正在学Java，我就试着分析一下。

原问题：一道java面试题

关于程序员对 Java 类型系统的理解，比较高级的一个面试问题是这样：

public static void f() {
    String[] a = new String[2];
    Object[] b = a;
    a[0] = "hi";
    b[1] = Integer.valueOf(42);
}

这段代码里面到底哪一行错了？为什么？如果某个 Java 版本能顺利运行这段代码，那么如何让这个错误暴露得更致命一些？

注意这里所谓的「错了」是本质上，原理上的，而不一定是 Java 编译器，IDE 或者运行时报给你的。也就是说，你用的 Java 实现，IDE 都可能是错的，没找对真正错误的地方，或者没告诉你真正的原因。

如果你知道哪里错了，并且知道「为什么」错了，可以联系我。

错在b[1] = Integer.valueOf(42);语句尝试往专门为String类开辟的空间放置Integer对象。

首先我试着修改一下代码，：

public class ForTest {
    public static void main(String[] args) {
        f();
    }
    public static void f() {
        String[] a = new String[2];
        Object[] b = a;
        a[0] = "hi";
        b[1] = Integer.valueOf(42);
        for (String e:
             a) {
            System.out.println(Arrays.toString(a));
        }
    }

}

运行时抛出异常：java.lang.ArrayStoreException: java.lang.Integer。

An array is a data structure that stores a collection of values of the same type。

根据数组的定义来看，你在尝试往数组里装不同类型的数据，所以抛出异常。

但我们来仔细看看代码：

String[] a = new String[2];
Object[] b = a;
a[0] = "hi";
b[1] = Integer.valueOf(42);

上面共四行代码：

  1. 第一行声明了一个String类型的数组；

  2. 第二行尝试将子类（String）数组引用转换成父类数组引用，在Java中这么做是合法的，而且不用强制转换；(In Java, arrays of subclass references can be converted to arrays of superclass references without a cast.—-出自core java volumn 1 Fundamentals，11th edition (Polymorphism section))

  3. 第三行将字符串"hi"放到String数组a里面，没毛病。

  4. 关键在第四行：虽然在编译器看来，数组b是Object数组，所以可以存放子类Integer的instances。但实际上数组b指向的内存本来是专门为String类开辟的，你往里面存放Integer实例，可能会导致“内存错乱”，比如你访问一个不存在的方法，就可能导致你的访问超出“本来数组a指向的那块空间”。

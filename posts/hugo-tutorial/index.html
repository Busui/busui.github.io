Hugo教程系列（一）

Hugo∣ 教程∣   Hugo教程∣   2019-08-14  2056字

一些说明

本教程不是从零开始教你写网站，需要你懂一些预备知识：

  1. 了解常见的终端命令；
  2. 了解git；
  3. 了解基础的css/html知识；

如果你先前有过部署hexo网站的经验，这个教程会使得你平滑地进入Hugo的世界。

我十分希望把教程写得简洁废话少，但每个人的情况不一样，我是把讲清楚放到第一位的。所以有些内容可能会显得唠叨。见谅！

Hugo教程系列主要介绍Hugo框架的使用方法，涉及内容有Content Management,Templates, Functions, Variables.

同时，这个系列会从头到尾教会你编写一个真正属于你自己的主题。为了保持内容正确性，全程参考官方文档。

快速开始

以本站主题yellow为例，快速构建出一个和本站一模一样的博客网站。

注意：本系列所有文章以macOS系统为例。如果你是其他系统，请自己参考官方安装教程。

在开始之前，希望你先完成Git安装。

步骤一：安装Hugo

Homebrew是macOS上的一款相当好用的包管理器。如果你还没有安装它，可以在Homebrew官网上快速找到安装方法。

使用brew安装Hugo：

brew install hugo

输入下面命令验证你的安装：

hugo version

步骤二：创建一个新的网站

hugo new site quickstart

上面的命令会在你当前目录下创建一个名为quickstart的新文件夹。quickstart就是你的网站的根目录(根目录是哪里是十分重要的概念)。此后的所有故事，都发生在这个目录里面。

步骤三：添加主题

本教程采用yellow主题，如果你想查看其他可用主题，可以在官方主题上找到。

在安装主题之前，请确保你已经进入了你创建的网站根目录，在这里，我创建的网站根目录是quickstart：

cd quickstart

之后使用git直接克隆主题到对应的themes目录下：

git clone git@github.com:Busui/gohugo-theme-yellow.git themes/yellow

步骤四：复制配置文件

如果你前面都操作正确，那么现在你就发现有两个文件：

一个是quickstart/config.toml，另一个是quickstart/themes/yellow/theme.toml。

接下来，你只需要把config.toml的内容先清空，然后将theme.toml的内容复制到config.toml里。

步骤五：测试主题

到了这里，其实你已经将你的网站建好了。不信？敲一下下面的命令看看：

hugo server -D

你应该能在这条命令下的输出信息看到一个网址，一般是http://localhost:1313。在浏览器访问它，就可以看到一个十分简单的页面：

你可以尝试点击这个网站看看里面的各个页面。现在它很简陋，什么都没有，莫要慌张，你还没有创建文章呢。

步骤六：创建文章

创建文章也是无比简单，只需要一个命令：

hugo new "posts/my first post.md"

到了这里，quickstart/content/posts目录下会有一个名为my first post.md的新文件。用你喜欢的markdown编辑器，编写文章吧。如果你还不懂markdown语法，百度一下，10分钟内你就能学会怎么用markdown语法写文章。

如果目前还不想学习markdown语法，或者想马上看到自己新建文章的样子，那么你可以将下面的内容复制追加到my first post.md文件里（注意，my first post.md本来已经有一些meta数据，所以这里是追加）：

### 下载Hugo

```
brew install hugo
```

### 创建网站

```
hugo new site quickstart
```

### 下载主题

```
cd quickstart && git clone git@github.com:Busui/gohugo-theme-yellow.git themes/yellow
```

### 创建文章

先将config.toml配置好。

然后输入下面命令：

```
hugo new "posts/your new post.md"
```

### 启动网站

```
hugo server -D
```


如果刚才你没有将hugo关闭，那么现在只要你保存文件，再到浏览器查看，马上就可以看到你刚写的文章。如果你已经关闭了，先保存文件，再用hugo server -D命令重新开启Hugo服务器。（你要注意命令的输出，不一定每次端口都是1313）

hugo server -D开启服务器后，除非你自己关闭，否则是一直运行着的。hugo服务器会一直关注着某些重要的文件的变化（需要你按保存，才能监测到），当这些文件发生变化时，hugo服务器能够捕捉到，然后通知浏览器作出对应的更新。

如果你不知道选用哪个编辑器，那么这里推荐你使用vscode。事实上，本网站所有的东西都是在vscode里面完成编写的。

步骤七： 更新主题

不管主题是自己写得还是用别人的，所谓更新主题，就是将最新的主题下载下来，替换掉你themes目录下对于旧的主题。所以说更换主题的方式是多种多样的。

这里我们使用的是最常见的方法，用版本控制软件：git。只需要进入到themes/yellow目录下，在命令行下输入：

git pull

一步搞定。

接下来怎么办？

到了这里。实际上你已经完成了你网站的建设了。如果没什么意外，整个过程不到20分钟就完成了。是不是有点惊讶，20分钟就可以搭建出一个相当不错（要看你安装的主题）的网站出来。

对于一般的内容工作者来说，接下来的步骤就是发布网站。然后写文章，更新网站，写文章，更新网站……

所以，如果你是一般内容工作者，可以直接阅读Hugo教程系列（十一）。

如果你想真正运营好你的网站，你就得进一步了解Hugo。比如理解Hugo的工作原理，理解函数和变量，然后，你就可以自己开发出一个属于你自己的主题。这也是本系列教程接下来要说的内容。继续吧！

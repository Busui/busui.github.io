此刻，如果访问的你的网站，就会发现，除了“关于”页面外，每个页面都可以访问了。（因为在config.toml的[taxonomies]中，你没有把about写上去。）你迫不及待地点击每个页面，发现每个页面都有相同的header和footer，因为它们继承于同一个baseof.html。但很快你就失望了，因为它们都长得一样。内容里只有一句话：“this is list page”。

1. 完善list.html

我们现在知道，smalltheme主题里的“分类/标签/归档”页面都是使用list.html模板。（注意：复杂的主题基本会为每个页面都设置单独的模板）

现在要解决的难题是：“分类/标签/归档”里面虽然都是list page（对于我们来说也就是文章列表），但这些列表的内容基本上都是不一样的，怎么使得同一个list.html模板，能够完美地为多个不同page（页面）服务呢？

答案是通过functions（函数）和varialbles（变量）。其实各种类型的页面都会有其对应的模板函数/变量：

  1. List Page Templates;
  2. Homepage Templates;
  3. Section Templates;
  4. Taxonomy Templates;
  5. Single Page Templates：这种稍后再说。

我们反复强调，其实Home pages，Section Pages，Taxonomy List Pages，Taxonomy Terms Pages都属于List Page。因此，解决方案也会呼之欲出了：

我们可以在list.html中调用“Home pages,Section Pages，Taxonomy List Pages，Taxonomy Terms Pages”对应的模板函数和变量

现在，我们便来一步步看list.html如何写吧。Hugo模板是渲染content目录下的各种posts的。所以开始之前，我们需要现在content目录创建一些posts，我已经将这些posts准备好了，通过git可快速下载：

1-2. 完善页面的内容布局

在官方找到你需要的页面或者函数，然后一点点调试，现在，我们的list.html内容如下：

            <div class="col-4"><span class="float-right">{{ .Date.Format "2006-01-02" }}</span></div>

2. 待解决的问题

如果你认认真真在浏览器浏览目前smalltheme主题的各个部分，你会发现还剩下一些问题：

  1. 归档页面没有显示出文章列表；
  2. “关于”页面空白；
  3. 无法查看特定文章的内容：比如你现在无法在浏览器看到“第一篇文章“的内容；
  4. 404.html为何物？怎么写？

我们接下来，就一步步解决上面的四个问题吧！

2-1. 让归档页面燃起来！

此时此刻，如果点击“归档页面”，你会得到下面的结果：

为什么没有显示任何文章呢？答案是Hugo还不知道“归档页面”到底该显示什么文章。默认下，Hugo知道在tags页面下显示的是tags相关的文章，categories页面下显示的是categories相关的文章。

怎么办呢？自然是我们告诉Hugo，“归档”这个页面到底该显示什么文章咯。

归档这个页面的作用是显示我们content目录下的所有post文章。在hugo里，RegularPages是post（文章）的集合，因此，我们可以利用.Site.RegularPages这个十分有用的变量来构建我们的“归档页面了”！

list.html内容如下：

{{ define "main" }}
<main>
    <span class="d-flex p-4 font-weight-bold">
        {{ if eq .Title "Categories" }}
            本站共有{{ len .Site.Taxonomies.categories }}个分类
        {{ else if eq .Title "Tags" }}
            本站共有{{ len .Site.Taxonomies.tags }}个标签
        {{ else }}
            {{ .CurrentSection.Title }}&nbsp;-->&nbsp;“{{ .Title }}”下的文章
        {{ end }}
    </span>
    <section class="pt-4">
        {{ if eq .Title "Archive" }}
            {{ range .Site.RegularPages }}
                <article class="row justify-content-between w-75 mx-auto mb-1 alert alert-success shadow" role="alert">
                    <div class="col-8"><a href="{{ .RelPermalink }}">{{ .Title }}</a></div>
                    <div class="col-4"><span class="float-right">{{ .Date.Format "2006-01-02" }}</span></div>
                </article>
            {{ end }}
        {{ else }}
            {{ range .Pages }}
            <article class="row justify-content-between w-75 mx-auto mb-1 alert alert-danger" role="alert">
                <div class="col-8"><a href="{{ .RelPermalink }}">{{ .Title }}</a></div>
                <div class="col-4"><span class="float-right">{{ .Date.Format "2006-01-02" }}</span></div>
            </article>
            {{ end }}
        {{ end }}
    </section>
</main>
{{ end }}

归档页面也成功渲染了：

2-2. 构建“关于页面”

到目前位置，“关于页面”点击还是空白的。事实上，想让“关于页面”能够被渲染出来是很容易的，因为“关于页面”是一个section，换句话说它是属于“list page”，所以只需要在config.toml添加点东西：

[taxonomies]
  archive = "archive"
  categories = "categories"
  tags = "tags"
  about = "about"   # 增加这行

再访问about时如下：


但很明显，我们的目的不是这样。我们希望“关于页面”是显示博主个人信息的页面。因此，和HomePage一样，我们要为“关于页面”单独写一个”list.html”。

那layouts下不就有两个list.html了？

是的，所以我们要遵守一些规则：

“把这个list.html放在smalltheme/layouts/about/目录下，这样，这个list.html就只是对about这个section有效了。”

再次提醒，content目录下的顶层文件夹都默认是section，例如content/about/这个文件夹默认下就是一个名称为about的section。

如下，我们先创建需要的文件/文件夹，我们希望现在你的content目录结构是这样的：

~/quickstart

├── 第10篇文章.md
├── 第3篇文章.md
├── 第4篇文章.md
├── 第5篇文章.md
├── 第6篇文章.md
├── 第7篇文章.md
├── 第8篇文章.md
├── 第9篇文章.md
├── 第一篇文章.md
└── 第二篇文章.md

同时我们希望你的layouts目录结构是这样的：

~/quickstart 
➜ tree themes/smalltheme/layouts
themes/smalltheme/layouts
├── 404.html
├── _default
│   ├── baseof.html
│   ├── list.html
│   └── single.html

│   └── list.html           # 请新建list.html文件
├── index.html
└── partials
    ├── footer.html
    ├── head.html
    └── header.html

到了这里，你应该知道layouts/about/list.html只是对“关于页面”页面有效。（ps：你可以通过front matter的配置，使得它对其他页面也有效）

现在我们看一下content里的内容，其中：

└─ about                   
   └── _index.md

_index.md的作用是什么呢？

官方文档里面说：

_index.md has a special role in Hugo. It allows you to add front matter and content to your list templates.

翻译过来就是：

_index.html在Hugo扮演了一个特别的角色：它允许你在list templates中添加front matter和content。

换句话说，也就是你可以在_index.md里面写任何东西（就像平时写文章一样），然后，在list.html模板中我们可以获取你写的这些东西，从而达到你可以像编辑文章一样随时编辑你的“关于页面”。

注意: 我们上面提到的.Site.RegularPages是获取不到_index.md的。因为_index.md不属于regular page。

2-3. 让文章可访问

其实我们已经提了很多次了，展示文章的页面属于single page（regular page），因此，只需要把smalltheme/layouts/_default/single.html写好就好了。

single.html内容如下：

{{ define "main" }}
<main class="container-fluid px-1 px-sm-3">
  <article id="main">
    <h4 class="border-bottom p-2 m-auto" id="title">{{ .Title }}</h4>
    <div class="row justify-content-end mx-2 mx-sm-4">
        {{ with .Params.tags }}
          标签：<span>
              {{ range . }}
              <a href="{{ "tags" | absURL }}/{{ replace . " " "-" | lower }}/">{{ . }}</a>&shortmid;
              {{ end }}
          </span>&nbsp;
        {{ end }}
        类别：{{ with .Params.categories }}
          <span>
              {{ range . }}
              <a href="{{ "categories" | absURL }}/">{{ . }}</a>&shortmid;
              {{ end }}
          </span>&nbsp;
        {{ end }}
      <span>时间：
        {{- .Date.Format "2006-01-02" -}}
      </span>&nbsp;
      <span>
          {{- .WordCount -}} 字
      </span>
    </div>
    <br>
    <div class="row">
      <div class="col-12 px-2 px-md-3" style="font-size: 1.2rem;">
          {{ .Content }}
      </div>
    </div>
  </article>
  <aside id="meta">
    <div class="row justify-content-between py-2 py-md-3">
        {{ with .PrevInSection }}
        <div class="col-12 col-md-5">
            上一篇<a href="{{.Permalink}}"> {{.Title}}</a>
          </span>
        </div>
        {{ end }}
        {{ with .NextInSection }}
            {{ if $.PrevInSection }}
                <div class="col-12 col-md-5">
                    <span class="float-md-right">
            {{ else }}
                <div class="col-12">
                <span>
            {{ end }}
            下一篇
            <a href="{{.Permalink}}"> {{.Title}}</a>
          </span>
        </div>
        {{ end }}
    </div>
  </aside>
</main>
{{ end }}

single.html弄了些花里花俏的东西。但重点不外乎下面几个变量：

  1. {{ .Content }}: 获取文章主要内容。
  2. {{ .Title }}: 获取front matter中的标题。
  3. {{ .Permalink }}: 获取文章链接。

2-4. 404.html

如果你没有忘记，在没有实现single.html的时候，如果你访问任何一篇文章，都会看到下面提示：

404 page not found.

这是因为你访问的页面不存在。因此，404.html的作用就是，当用户访问的页面不存在时，就展示404.html给用户。我们来看一下404.html里面的内容:

{{ define "main" }}
<div class="container">
    <h4 class="align-center">对不起，你访问的页面不存在</h4>
</div>
{{ end }}

由于我们现在是在测试阶段，没有开启web服务器，也没有托管到github上，所以只能手动访问这个页面了。（ps：到时候发布网站，作出相关配置时，访问不存在网页时，会自动跳转到404页面）

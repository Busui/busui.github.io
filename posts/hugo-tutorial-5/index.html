Hugo教程系列（五）

template∣ Hugo∣ 教程∣   Hugo教程∣   2019-08-30  5314字

Hugo使用的是go模板，但只保留了最精髓的功能。学会了go模板的相关内容后，你就可以开始编写主题了。

关于template的介绍内容实在有点多。我把它们分成3部分，这是“template介绍的第一部分”

1. 基础语法

Go模板其实是html文件，但和普通的html文件不同，在Go模板中我们可以使用variables（变量）和调用functions（函数）。一般来说，写在{{...}}里的内容会被认为是variables或者functions。

1-1. scope（作用域）

首先，什么变量？变量就是一个名称，通过这个名称，我们可以访问到变量指向的常量（实际上变量指向的是一个地址，但这里只需要你明白，通过变量，我们可以得到变量指向的“东西”）：

{{ $valiable := 1}}

$valiable是变量，通过它，我们可以得到常量1。

说到了变量，那么作用域是一个不得不谈的话题。

那什么是作用域呢？作用域，其实就是一个变量的有效范围。这里，要看到本质的东西：

  1. 作用域是针对变量来说的；
  2. 作用域是范围。

那为何要给变量定一个范围呢？

对变量来说，范围的作用就是为了管理变量。让变量呆在该呆的范围之内。为什么要这样呢？我们可以从我们平时开发的角度来说。一般来说，一个项目是多人开发的，这可能会造成一种糟糕的情况：两个人起了相同的变量名。

比如帅哥A将变量balance的值设为1000，帅哥B却将balance的值设为-100000，那编译的时候，balance的值无论选取哪个都会酿成错误。

但如果有作用域来限制变量，那么根据作用域，编译器就可以智能地识别balance的值，就可以使得程序正确运行了。

那么，在go模板里的作用域如何理解呢？其实上面说过了。很多时候，我们会遇到变量名相同的情况，作用域的主要目的是让这些同名的变量在各自的作用域里各司其职，互不影响，互不干预。

现阶段，我们只要记住两种作用域：

  1. 不同的go模板就是不同的作用域,不同的content+同一个go模板也会形成不同的作用域；

  2. 有时候在一个go模板内，有些函数会产生需要被“圈起来”的结果，圈起来的范围，也会形成一个独立的作用域。如果非要总结点规律出来（这种总结是不科学的，但目前你需要有这么一个概念），就是这样：

    {{ Function arg1 arg2 ... }}
被包起来的这个区域会形成独立的作用域
{{ end }}

再次强调，作用域的主要目的是让那些同名的变量在各自的作用域里各司其职，互不影响，互不干预。也可以换句话讲：防止同名变量互相干扰，引起冲突。

额。如果你还是看不懂什么是作用域，什么是变量，什么是常量，建议自己搜索一下。因为如果这些东西都得展开讲，那么这个教程就变味了。

1-2. Access a Predefined Variable（访问预定义变量）

在go模板中，有预定义变量这么一种概念。预定义的变量有两种：

  1. 已经存在于当前作用域的某个变量，这些变量是Hugo内部实现的，开箱即用。（比如下面例子的.Title变量）；

    {{ .Title }}

  2. 你自己定义的变量。（比如下面例子的$address变量）。

    {{ $address }}

函数的参数是以空格隔开的。下面是基本语法：

{{ FUNCTION ARG1 ARG2 .. }}

我们可以马上举例。add函数接收两个参数作为输入。比如计算1+2:

{{ add 1 2 }}

1-3. Methods and Fields are Accessed via dot Notation

methods（方法）和Fields（字段）是通过.符号语法来访问的。比如我的文章front matter字段如下：

title: "Hugo教程系列（五）"
date: 2019-08-30T10:08:29+08:00
bar: test

如果我要获取bar的内容，那么我可以通过.符号语法来访问到：

{{ .Params.bar }}

1-4. Parentheses Can be Used to Group Items Together

括号可以用来将一些表达式组成一组，先算括号里的内容。换句话讲，就是遇到括号，先算括号里的：

{{ if or (isset .Params "alt") (isset .Params "caption") }} Caption {{ end }}

2. Variable(变量)

每当输入hugo命令并按下回车时，layouts目录下的每一个go模板都会自动获取一个数据对象。在Hugo里，每个模板都会被传递一个Page对象（可以这么理解，Page对象就是数据对象的其中一种）。每一个Page对象默认包含了许多Page变量。在下面的例子中，.Title就是其中一个可访问的Page变量。

本来我们是这样访问Title变量的：

<title>{{ .Page.Title }}</title>

但因为Page对象默认被传入当前模板的作用域内，所以我们可以省略掉.Page,直接通过.Title访问：

<title>{{ .Title }}</title>
这里能不能联想到，每一个不同作用域的模板都有相同的一个.Title变量，但这个变量在不同的作用域中它的值是不同的？也就是说，不同作用域中，相同名称的.Title变量互不影响，各司其职。

当然，我们也可以预先将值存放在自定义的变量里，然后在同一个作用域内，我们可以通过自定义的变量获取对应的值：

{{ $address := "123 Main St." }}
{{ $address }}  # 这个语句执行结果是"123 Main St."

以$为前缀的就是自定义的值。

在Hugo v0.47以及之前的版本，在if条件语句作用域内定义的变量和作用域外面是互相独立的，如下面代码：

{{ $var := "Hugo Page" }}
{{ if .IsHome }}
    {{ $var = "Hugo Home" }}
{{ end }}
Var is {{ $var }}

无论你访问哪个页面，上面的代码都是输出Hugo Page。因为{{ $var}}无法被重写。

但Hugo v0.48及以后的版本，我们可以用=操作符对变量重新定义：

{{ $var := "Hugo Page" }}
{{ if .IsHome }}
    {{ $var = "Hugo Home" }}
{{ end }}
Var is {{ $var }}

如果你当前访问的是homepage,那么上面的代码就输出Var is Hugo Home。

否则就输出Var is Hugo Page。

到了这里，是不是对“不同作用域的同名变量互不干扰，互不影响”产生疑问？其实概念是没错的。

只不过是出于某种原因，在某种情况下，Hugo开发者赋予了被重定义的能力给变量。你也可以这么理解吧，给这种情况开了绿灯。这种情况中我们也可以得出一个十分重要的道理：

“规则是人定的。”

没错，.Title, .WordCount, .Content都是变量。template就是通过这些变量，获取到content目录里对应section下对应文章里面的标题，文章字数，文章内容:

变量的威力估计你已经能体会了。就是通过这么小小的一个变量，就能从content目录下各种文件里面获取到我们想要的任何“东西”。

这就完事了？非也。你再看看，content/posts目录下的3篇文章其实都是由single.html这个模板渲染出来的。也就是说，single.html模板里面的变量（.Title, .WordCount,.Content …），它们在渲染不同的文章时，它们的值是不一样的。这怎么做到的呢？

答案是：通过scope（作用域）/Context。

提醒：作用域是针对变量而言的。context（上下文）将会在下一篇文章说到，威力更加巨大。

我们在前面就说过：

  1. 不同的content+同一个go模板也会形成不同的作用域；

  2. 作用域的主要目的是让那些同名的变量在各自的作用域里各司其职，互不影响，互不干预。

看见了没有，这，就是作用域的威力。

有的同学可能会有疑问，既然{{...}}里面的内容可能是变量，也可能是函数，那我怎么判断里面是变量还是函数？额，一般来说，查官网是最稳妥的办法。其次，工多手熟罢了。

4. Function（函数）

Go模板只是提供了最基础的函数集，但同时也提供扩展这些函数集的机制。

为了让你能更好地建设网站，Hugo模板函数提供了额外的一些十分有用的函数（注意，是Hugo模板函数！！！这是Hugo在go模板基础上，自己添加的函数）。我们前面就已经见到，通过函数的名称来调用函数，它们的参数则放在函数名称后面，多个参数则用空格隔开。除非重新编译Hugo，否则模板函数不能被添加。

4-1. Example 1: 数字相加

{{ add 1 2 }}
<!-- 输出 3 -->

4-2. Example 2: 比较数字

{{ lt 1 2 }}
<!-- lt，是less than的意思，就是小于。因为1小于2，所以上面语句输出是true -->

5. include 包含/导入

如果你学过编程，大体上都能知道何谓include。在Hugo中，所说的include一般就是指导入其他模板。有两个函数来做这件事儿：

  1. partial：用来导入用户自己编写的partial模板；
  2. template：用来导入Hugo内置的partial模板。

5-1. Partial函数

partial是用来在当前的模板导入其他模板（即partial模板）的函数。partial模板文件一般放在layouts/partials/目录下。

事实上在前端开发时，为了使得开发变得更清晰，会将一些能放在一起的，而且可以重复使用的代码片段放在独立的文件当中，之后就使用partial函数来“包含这些文件”，这样就达到将该文件组装到当前的文件里面的目的。

例子：在某模板里面包含layouts/partials/header.html这个partial:

{{ partial "header.html" . }}
不知道你有没有发现，”header.html”后面跟着一个.，这个.在Hugo里面可大有含义。它代表将当前模板的上下文（也就是环境）传递过去。这样”header.html”就能知道自己是被哪个模板导入了，而且在”header.html”也能使用当前模板作用域内的所有东西。

5-2. Template函数

在以前的Hugo版本中，template函数的作用和partial函数的作用是一样的。但目前template函数只是用来将内置的partial文件导入当前模板。

再说一次，除了你自己写的partial模板，Hugo本身也内置了许多Internal Templates(内置模板)。

例子：将Google Analytics(谷歌分析)导入当前模板：

{{ template "_internal/google_analytics.html" . }}

数据结构开篇

数据结构开篇∣   数据结构∣   2020-02-19  1847字

维基百科里对数据结构的定义如下：

In computer science, a data structure is a data organization, management, and storage format that enables efficient access and modification. More precisely, a data structure is a collection of data values, the relationships among them, and the functions or operations that can be applied to the data.

虽然不提倡把这个定义死记硬背下来，但一定要意识到在这个定义里的信息量之巨大，非常值得我们花时间慢慢分析出这句话的含义。

首先，分析第一句话：

In computer science(说明我们讨论的数据结构是限定在计算机科学这个领域的), a data structure is a data organization, management, and storage format that enables efficient access and modification. (这里提到数据结构这个学科研究的主要内容：1. 组织数据， 2. 管理数据， 3. 一个能够高效访问/修改数据的存储格式)。

至此，我们可以小小归纳一番，数据结构是一个学科，这个学科研究的主要内容是：

  1. 如何组织数据？
  2. 怎样管理数据？
  3. 在内存中使用何种存储格式才能使得程序能够高效访问/修改数据？

数据从哪里来，到哪里去？

我们在学习数据结构的时候，一直在说数据，那么我们不妨问一下自己：（在计算机科学里）数据到底是哪里来的，数据最终又到哪里去？其实有两条路径：

  1. 从人类世界来，到计算机世界里去；
  2. 从计算机世界来，到人类世界中去；

因为人类和计算机是完全不同的物种，计算机是只懂二进制的，而面对海量的二进制时，人类本身是无能为力的（我们对二进制不敏感，甚至大多数人不懂二进制），因此需要使用一系列的操作使得人类的数据最终能够存放到计算机里（反过来说，计算机存放的数据最终能被人类利用）。我用下图大致描绘一下这个过程：（ps：为了简单化，下图只给出了数据从人类世界到计算机世界的大致过程）

当然，上面阐述的内容不太专业，下面是更加专业的说法：

数据结构的研究内容是构造复杂软件系统的基础，它的核心技术是分解与抽象。通过分解可以划分出数据的3个层次,再通过抽象，舍弃数据元素的具体内容，就得到逻辑结构。类似地，通过分解将处理要求划分成各种功能，再通过抽象舍弃实现细节，就得到运算的定义。上述两个方面的结合可以将问题变换为数据结构。这是一个从具体（即具体问题）到抽象（即数据结构）的过程。然后，通过增加对实现细节的考虑进一步得到存储结构和实现运算，从而完成设计任务。这是一个从抽象（即数据结构）到具体（即具体实现）的过程。

我们先提取重要概念（我们希望你好好看下面的两段话）：

  1. 从具体（即现实世界存在的具体问题）到抽象（即数据结构）的过程：通过分解可以划分出数据的3个层次,再通过抽象，舍弃数据元素的具体内容，就得到逻辑结构。类似地，通过分解将处理要求划分成各种功能，再通过抽象舍弃实现细节，就得到运算的定义。上述两个方面的结合可以将问题变换为数据结构。
  2. 从抽象（即数据结构）到具体（即计算机世界里的具体实现）的过程：通过增加对实现细节的考虑进一步得到存储结构和实现运算，从而完成设计任务。

接下来，要对上面两点分阶段讨论。

逻辑结构

从具体（即现实世界存在的具体问题）到抽象（即数据结构）的过程中，我们需要找出数据元素之间的逻辑关系，得到逻辑结构。

逻辑结构有两大类：

  1. 线性结构：数据结构中的元素存在一对一的相互关系；
  2. 非线性结构：数据结构中的元素不是存在一对一的相互关系。

存储结构（又称物理结构）

从抽象（即数据结构）到具体（即计算机世界里的具体实现）的过程中，我们会得到具体的运算和存储结构。

具体的运算是和算法相关的。

而存储结构则有两大类：

  1. 顺序存储结构：借助元素在存储器中的相对位置来表示数据元素之间的逻辑关系；
  2. 非顺序存储结构：借助指示元素存储地址的指针表示数据元素之间的逻辑关系。

逻辑结构和存储结构的联系

以前学习数据结构的时候，总以为数据结构里的逻辑结构和存储结构只是按不同的方向去分类而已，它们本身是没什么联系的。随着时间推移，我才意识到，其实逻辑结构和存储结构是紧密联系在一起的，它们是编写程序时的不同阶段：

我们从现实世界得到数据的时候，根据这些数据之间的联系，我们会先得到逻辑结构，然后根据逻辑结构，我们会得到恰当的存储结构。

比如：我们有一个银行的排队系统需要完成，我们现对银行里排队业务进行分析，因为一个人一个人轮着来，所以很容易确定应该是线性结构，接下来我们分析到数据元素（人）之间的联系是“先进先出”，因此我们将队列作为我们的逻辑结构（此时得到了逻辑结构），之后对队列这个逻辑结构进行分析：

  1. 如果觉得它该使用顺序存储结构，我们就选用数组模拟队列；
  2. 如果觉得它该使用非顺序存储结构，我们就选用指针。

数据的逻辑结构和物理结构是数据结构的两个密切相关的方面，同一逻辑结构可以对应不同的存储结构。算法的设计取决于数据的逻辑结构，而算法的实现依赖于指定的存储结构。

1. Logic （逻辑）

Go模板提供最基础的Iteration（迭代）逻辑和conditional（条件）逻辑。

1-1. Iteration（迭代）

所谓迭代，其实就是对某个可迭代的集合按某种顺序逐一输出。在go模板中，可迭代的集合有三种：

  1. map（映射）；
  2. array（数组）；
  3. slice（切片）。

ps：如果你学过python，那么上面的内容你绝对不会陌生。没学过也没事，请继续看。

可以说，range是Go模板里最常用的函数了。它的作用是对可迭代集合进行取出操作。

Iteration（迭代）下面的五个例子只是为了演示某函数/某变量背后的原理。语法层面上来说，这些例子在Go模板中是运行不通的。

例子 1: Using Context (.) （使用上下文）:

我们假设$array数组的内容是[“one”, “two”, “three”]：

{{ range $array }}
    {{ . }} <!-- `.` 代表$array数组中的一个成员 -->
{{ end }}

上面代码输出大概是：

"one"
"two"
"three"

如果$array里面的内容是一篇篇文章，那么上面短短几行代码，就可以输出$array所有的文章。下面这种文章排列的效果就是range函数的功劳：

例子 2: 为每个array数组成员的值声明一个变量名：

{{ range $elem_val := $array }}
    {{ $elem_val }}
{{ end }}

输出和例子1一样

例子 3: 为每个array成员的index(索引)和value(值)分别声明变量名

对于array和slice来说，声明的第一个变量是指向其成员的index(索引)

{{ range $elem_index, $elem_val := $array }}
   {{ $elem_index }} -- {{ $elem_val }}
{{ end }}

上面代码的输出大概是：

0 -- "one"
1 -- "two"
2 -- "three"

例子 4: 为每个map的key（键）和value（值）分别声明一个变量名：

对于map来说，声明的第一个变量名会指向其成员的key（键）。

{{ range $elem_key, $elem_val := $map }}
   {{ $elem_key }} -- {{ $elem_val }}
{{ end }}

例子 5: map, array, 或者slice内容为空时的操作：

{{ range $array }}
    {{ . }}
{{else}}
    <!-- 当$array内容为空时，此处的代码将被执行 -->
{{ end }}

1-2. Conditionals (条件语句)

在Go模板中中，使用if,else,with,和and实现条件语句逻辑。就像range，每一个语句后都需要追加{{ end }}以代表结束。

Go模板认为下面的值为false:

  1. false(boolean布尔值)；
  2. 0（整数）；
  3. 任何长度为0的array,slice,map或者string。

例子 1: with :

使用with来表达“如果某事为真，则做……”逻辑，是十分普遍的事。

就像range一样，with也会在使用.符号来绑定它自己的上下文。关于什么是上下文，文章下面会提到。

如果.Params.title的值为空，则不执行with和end之间的语句块（即不执行<h4>{{ . }}</h4>），反之，则执行。

{{ with .Params.title }}
    <h4>{{ . }}</h4>
{{ end }}

例子 2: with .. else:

下面代码片段的功能是：如果在文章的front matter已经设置description的值，则使用这个description的值，否则使用默认的{{ .Summary }}:

{{ with .Param "description" }}
    {{ . }}
{{ else }}
    {{ .Summary }}
{{ end }}

.Summary变量是Hugo内置变量，功能是截取你写的文章开头一部分来作为文章的简介。

而关于.Param,请看.Param function。

例子 3: if：

实现条件语句逻辑的一个备用方案是使用if语句。但if语句没有绑定 . 上下文。所以说，with语句实际上是if语句的升级版本。

下面的例子实现了和“例子 1”一模一样的效果。很明显，使用if语句代码量多了。

{{ if isset .Params "title" }}
    <h4>{{ index .Params "title" }}</h4>
{{ end }}

例子 4: if .. else：

下面例子实现了和“例子 2”一模一样的效果。

{{ if .Param "description" }}
    {{ index .Params "description" }}
{{ else }}
    {{ .Summary }}
{{ end }}

例子 5: if .. else if .. else：

和with函数不同，if语句可以使用else if语句：

{{ if (isset .Params "description") }}
    {{ index .Params "description" }}
{{ else if (isset .Params "summary") }}
    {{ index .Params "summary" }}
{{ else }}
    {{ .Summary }}
{{ end }}

例子 6: and & or：

{{ if (and (or (isset .Params "title") (isset .Params "caption")) (isset .Params "attr")) }}

关于isset，请看isset function。

2. Pipes（管道）

其实在Hugo教程系列二我们提过pipes（管道）这个概念。由于将来我们构建主题会多次用到管道，所以这里详细说一下所谓管道到底是个什么。

函数被执行的时候，会产生某些操作。我们将这些操作，叫“actions（动作）”。Go模板最强大的功能之一就是：能够将一个个动作压栈（可以暂时理解压栈就是存储）。这个功能是通过pipes（管道）来实现的。这个功能借鉴于Unix pipes，概念十分简单：每个管道的输出作为下一个管道的输入。

因为Go模板的语法相当简单，所以pipes必须要兼容这种简单，因此，pipe必须能够将函数调用链接（是链接，像锁链一样，环环相扣）起来。这么造成了一个限制：管道只能传递单个值。具体的链接结构如下：

管道只能传递单个值。可要好好理解这句话的意思。其实我觉得应该翻译成管道只能传递单个参数才对。

下面简单的例子应该能让你快速学会使用pipes：

例子 1: shuffle：

首先我们先来看seq函数的作用：

{{ seq 1 5}}    # 输出 [1 2 3 4 5]

再来看shuffle函数的作用：

{{ shuffle (seq 1 5) }}     #输出[4 3 2 1 5]

每次运行时，shuffle的输出结果都是不一样的。说明shuffle是将数组里面成员顺序打乱的函数。

接下来，别眨眼，看我们如何用管道来实现上面代码的功能：

{{ (seq 1 5) | shuffle }}

函数seq的执行结果，通过管道，作为shuffle的输入。

3. Context (aka “the dot”)

Context，中文意思是“上下文”，又名“the dot”。在Go模板中，最难理解的就是上下文。Go模板中的{{ . }}始终指向当前上下文。什么意思呢？就是说{{ . }}等于当前上下文。但，什么是当前上下文呢？

首先，什么是上下文？往大的方向去理解，上下文，就是使得某些事物/事件奏效的环境。怎么说呢？举个十分简单的例子吧：

“鱼游泳，有效范围是在水里。脱离了水这个环境，鱼还能游泳吗？所以说，在水这个上下文（环境）内，鱼游泳这件事儿才能奏效”

你仔细想想，其实我们也是活在某个上下文（环境）里面的。

那对template（模板）来说，上下文是什么呢？

额，上下文是环境。环境在哪？这个需要点想象力啊，因为环境是抽象的。hugo运行时，每个模板各自都会形成一个独立的环境。我们处理事情时一般会假设我们在某个上下文中，这个上下文，就是当前上下文。在每一个独立的上下文中，我们都调用所有Hugo支持的内置函数和内置变量。

更详细地说，Go模板形成的上下文其实有两种情况：

  1. 每一个模板文件都会形成各自的上下文（即环境），content目录下不同文件+同一个模板会形成不同的上下文。这种类型的上下文，是针对page（页面）的。或者说，是针对某个模板的，是全局的。
  2. 在迭代语句里面，循环时会形成自己独立的数据，因此，在迭代语句里面，会形成一个针对循环数据的上下文，这种上下文是局部的。

每一个上下文都会绑定一个数据集。这些数据集是一些内置函数和内置变量，或者是执行迭代语句产生的数据集。它们中很多函数/变量的名称都是相同的，但它们在不同的上下文中，会取到相应的值，各个相同名称的函数/变量不会互相冲突。

你会发现上下文和作用域的范围很像。其实在Hugo里，按照我看文档得出的结论，它们真的很像，甚至可以说是一样的（是说范围一样！！！）。但它们针对的对象是不一样的，作用域针对的是变量，上下文针对的是更大范围的函数和变量。


有这么一种情况，比如我们在局部上下文中想访问全局上下文里的变量，这么办？有两种方案，如下：

3-1. Define a Variable Independent of Context （定义一个独立于上下文的变量）

{{ $title := .Site.Title }}
<ul>
{{ range .Params.tags }}
    <li>
        <a href="/tags/{{ . | urlize }}">{{ . }}</a>
        - {{ $title }}
    </li>
{{ end }}
</ul>

在range语句里面，.Site.Title是不存在的。因为range语句里面形成的上下文并没有绑定.Site.Title这个变量。因此，我们在全局上下文中将.Site.Title的值存储在$title中，因为$title是独立于上下文的，所以在range语句内也能访问到它。

有一个十分要注意的地方是 {{ . }}代表的是当前上下文。也就是说，在全局环境中，{{ . }}代表的是“全局”上下文。而在range语句里，{{ . }}代表的是“局部”上下文。

3-2. Use $. to Access the Global Context

在局部上下文内，使用$.可以访问全局上下文。这种情况我们建立主题的时候没用到，因此需要了解更多，请查看Use $. to Access the Global Context。

4. Whitespace （空白）

在将go模板渲染成html文件时，会产生空格(假设.Title的值是Hello,world!)：

<div>
  {{ .Title }}
</div>

渲染成html是，会变成下面这样：

<div>
  Hello, World!
</div>

Go模板提供了消除空格的方法：

<div>
  {{- .Title -}}
</div>

渲染成html后是这样：

<div>Hello, World!</div>

Go模板能够消除的空白有下面四种：

  1. 空格；
  2. 换行；
  3. 回车；
  4. tab。

Hugo教程系列（十一）

Hugo∣ 教程∣ 部署网站∣   Hugo教程∣   2019-09-12  3176字

部署网站的一个很重要目的就是令得你的网站可以被所有互联网用户访问到（请自行忽略某种不可抗拒因素）。

由于Hugo网站是静态网站，基本上没涉及到什么web运行环境的部署，因此发布Hugo网站其实是相当简单的。Hugo Hosting里几乎谈到了所有部署Hugo网站的方式，我们这里主要谈其中的两种：

  1. 部署到github pages；
  2. 部署到某主机（以阿里云为例，其他的其实也一样）

1. 简单介绍github pages

如果此前你有Hexo的操作经验，那么这一小节的内容可能会令你觉得无聊。

Github Pages为我们提供了免费的空间，我们只需要简单的一个脚本（或几个步骤）就可以将我们的网站托管在Github的空间里，剩下的，Github Pages会帮你完成。

在部署之前，请保证你满足下面条件：

  1. 在你的电脑上至少有2.8G的硬盘剩余容量；
  2. 已经在Github上注册账号，如果你还没注册，请点击Signing up；
  3. 一个已经生成好的静态网站，如果你还没有，请先完成Hugo教程系列（一）。

有两种类型的Github Pages：

  1. User/Organization Pages：通过https://<USERNAME|ORGANIZATION>.github.io/访问
  2. Project Pages：通过https://<USERNAME|ORGANIZATION>.github.io/<PROJECT>/访问。

我们这里只是涉及到第一种。其中,USERNAME是你的github用户名。比如我的用户名是busui，那么把我们的网站上传之后，我就可以通过https://busui.github.io 访问我的网站。

2. 部署到Github Pages和网站更新

其实部署到Github Pages上，也就是将我们的静态网站上传到名为<USERNAME>.github.io的仓库里面。

由于我们每次写文章都是在电脑本地完成的，而且每次完成都会重新生成一个新的静态网站，因此，我们在往后还是会不断的将网站一次次上传到同一个仓库里。我们称这一过程是网站更新。

到了这里，你就要明白，使用Github Pages我们要做的工作就是：

  1. 部署网站
  2. 更新网站

部署和更新都是将网站上传到GitHub Pages上，那它们的区别是什么呢？额，部署网站是第一次上传网站，第一次嘛，肯定难点，要做的准备也多点。

好，下面我们就来看看到底部署网站有多“难”。

2-1. 创建新仓库

首先，第一步就能让你蒙圈。因为在这一步我们会创建两个仓库。（ps：当然，另一个仓库是可选的，但还是建议你创建两个）

在前面谈到Github Papes的时候我们反复强调，我们只是将生成的静态网站（也就是项目下的public文件夹）上传到名为<USERNAME>.github.io的仓库里。为了方便，我把这个仓库称作网站仓库。

其实，我们还得创建另一个仓库，这个仓库，作用是备份你的Hugo项目。为了方便，我把这个仓库称为项目仓库。这样，无论你走到天涯海角，只要你记得你的github账号密码，你就可以在很短时间内在电脑中恢复你的Hugo写作环境。

到了这里，我们先捋清一些思路：

  1. 项目仓库: 存放Hugo项目的，如在Hugo教程系列（一）我们创建的quickstart项目；
  2. 网站仓库: 存放public文件夹的，需要注意的是这个public文件夹在Hugo项目里面；
  3. 项目仓库包含网站仓库（别看这句话短，真正理解可不容易）😂

2-2. 更新网站

更新网站相对就简单很多了，先在config.toml设置对应的值：

baseURL = "https://busui.github.io"   # 把busui更改为你的github名称

# 剩下的值你自己酌情修改

之后每次写完文章，在网站根目录下，逐条输入下面的指令即可：

rm -rf public
hugo
cd public
git add .
git commit -m "这次更新的说明"
git push

每次输入上面的指令有点烦，有没有一条指令解决所有事情的呢？有！

我们先在项目根目录下创建一个名为deploy.sh的文件，然后在里面写入下面内容：

#!/bin/sh

# If a command fails then the deploy stops
set -e

printf "\033[0;32mDeploying updates to GitHub...\033[0m\n"

# Build the project.
hugo # if using a theme, replace with `hugo -t <YOURTHEME>`

# Go To Public folder
cd public

# Add changes to git.
git add .

# Commit changes.
msg="rebuilding site $(date)"
if [ -n "$*" ]; then
	msg="$*"
fi
git commit -m "$msg"

# Push source and build repos.
git push origin master

之后，每次写完文章，只需要在终端下（注意路径在项目根目录喜爱），输入：

./deploy

完美！

3. 部署到阿里云

这个其实是相当简单的。在购买云主机之后我们会得到一个主机密码。你可以创建一个普通用户，之后安装一个web服务器，比如Nginx。

之后购买域名，备案。比如我的域名是codepool.top。

最后写一个名为ali-deploy.sh的脚本：

#!/bin/sh
USER=在云主机上你的登录名
HOST=主机ip地址
DIR=www/   # might sometimes be empty!


# --baseURL的值改为你的域名
hugo --baseURL="https://codepool.top" && rsync -avz --delete public/ ${USER}@${HOST}:~/${DIR}

exit 0

每次更新是至于要在终端下输入：

./ali-deploy.sh

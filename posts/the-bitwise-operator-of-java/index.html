The Bitwise Operator of Java

位操作符∣   java∣   2020-02-23  703字

For any of the integer types, you have operators that can work directly with the bits that make up the integers. This means that you can use masking techniques to get at individual bits in a number. The bitwise operators are:

& (“and”) | (“or”) ^ (“xor”) ~ (“not”)

There are also >> and << operators which shift a bit pattern right or left. These operators are convenient when you need to build up bit patterns to do bit masking:

int fourthBitFromRight = (8 & (1 << 3)) >> 3;
System.out.println(fourthBitFromRight);
// output: 1

Finally, a >>> operator fills the top bits with zero, unlike >> which extends the sign bit into the top bits. There is no <<< operator.

对于正数来说，>>>和>>是没有区别的，因为无论它是用0来填充最高位，还是用符号位（正数的符号位是0）来填充最高位，都是一样的。

但对负数来数，>>>和>>有很大的区别的。如果是>>>, 那么就用0来填充最高位，如果是>>，那么就用符号位（即1）来填充最高位。如下：

byte myByte = -1;
System.out.println(">> operate on myByte:\t" + (myByte >> 1));
System.out.println(">>> operate on myByte:\t" + (myByte >>> 1));

int myInt = -1;
System.out.println(">> operate on myInt:\t" + (myInt >> 1));
System.out.println(">>> operate on myInt:\t" + (myInt >>> 1));

// output:
>> operate on myByte:	-1
>>> operate on myByte:	2147483647
>> operate on myInt:	-1
>>> operate on myInt:	2147483647

简单分析：

-1的补码是：1111 1111 1111 1111 1111 1111 1111 1111

对-1进行>>操作时，，最高位用符号位（即1）来填充，所以无论进行多少次>>操作，结果都是不变的。

对-1进行>>>操作时，最高位用0填充，进行>>>操作后，结果变成: 0111 1111 1111 1111 1111 1111 1111 1111(即2147483647)

下图有助于你理解上面的补码操作：

之所以分别用了byte和int类型来测试，是因为>>和>>>默认是用4个字节来运算。（PS：如果操作数是long，>>和>>>则用8个字节。）

The right-hand argument of the shift operators is reduced modulo 32 (unless the lefthand argument is a long, in which case the right-hand argument is reduced modulo 64). For example, the value of 1 << 35 is the same as 1 << 3 or 8.

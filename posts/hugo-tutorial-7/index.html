Hugo教程系列（七）

Hugo∣ 教程∣ template∣   Hugo教程∣   2019-09-02  1900字

在前面我们已经学习了模板的一些基础语法。这篇文章是template介绍的第三部分，接下来我们将介绍如何给代码添加注释，如何在模板中引用config.toml文件中的配置数据和front matter中的内容。

1. Comments （注释）

在开源项目postwoman中，作者说了这么一段话：

When I wrote this, only God and I understood what I was doing. Now, only God knows.

大意是：“当我写下这些代码时，只有上帝和我知道写了什么。现在，只有上帝知道了。” 虽然里头有调侃之意，但却写出了一个血淋淋的事实：很可能过了一段时间之后，连你自己都不知道当初写了些什么。

怎么办呢？注释，没错是注释。注释的作用是让别人或自己能够读懂自己写的代码。这在团队开发项目时尤为重要。当然，Go模板也是支持注释的。在Go模板中，有两种类型的注释，一种是针对Go模板本身的，一种则是针对HTML文件的。

1-1. Go Templates comments

针对Go模板的注释。在Go模板中，{{/*和*/}}之间的区域会被认为是注释的内容。Hugo不会渲染这个区域的内容。一个例子：

哈哈, {{/* {{ add 0 + 2 }} */}}呵呵.

Hugo会将上面的内容会输出为：

 哈哈, 呵呵.

按照我们学过的概念，{{ add 0 + 2 }}本该是会被Hugo执行的，但是！但是它现在在{{/*和*/}}之间的区域，因此，Hugo不会渲染它。（你可以这么认为吧，Hugo根本“看不见”{{ add 0 + 2 }}这条语句，根本不会在将它输出到html文件中，更别说执行它了。为什么Hugo“看不见”它，额，注释是写给Go模板开发者看的。）

1-2. HTML comments

针对HTML的注释。这种注释是写给浏览器/人看的，需要要内嵌在html。因此有些情况下你需要在你的模板中生成html注释，例如针对IE浏览器的条件语句注释。如果你需要在HTML注释中使用变量，只需要利用管道将变量内容作为safeHTML的输入。如下：

{{ printf "<!-- Our website is named: %s -->" .Site.Title | safeHTML }}

你会发现，针对Go模板的注释和针对HTML的注释有着天壤之别。

2. Hugo Parameters（Hugo 参数）

我们之前多次提到，Hugo模板能够获取在config.toml里面的网站配置信息，而且也能够获取每篇post的front matter里面的信息。

Site configuration（网站配置）可以存放在config.toml, theme.toml或者config/中。通过配置，甚至可以存放在任何地方。上面描述为了方便，就以config.toml作为例子。

2-1. Use Content (Page) Parameters（使用content参数）

如果以博客系统为参考点，所谓content Paramters，其实就是每篇post里面的front matter。以你正在看的文章的front matter为例子：

---
title: "Hugo教程系列（七）"
date: 2019-09-02T20:12:42+08:00
Author: Busui
tags:
    - Hugo
    - 教程
    - template
categories:
    - Hugo教程
sumary: hugo模板介绍第三部分
draft: false
summaryImg:
copyright: true
---

Go模板可以获取front matter里面的值，从而更好地渲染页面。怎么获取呢？如下：

<h1 id="title">{{ .Title }}</h1>        # 获取标题的值

{{ with .Params.tags }}                 # 获取tags里的内容
    {{ range . }}                       # tags的值可迭代。range函数后跟着的"."是代表with语句上下文。代表tags数组本身。
        {{ . }}                         # 这里的"."代表range语句里面包含的上下文。代表每一个tag成员。
    {{ end }}
{{ end }}

{{ with .Params.categories }}           # 获取categories里的内容
    {{ range . }}
        {{ . }}
    {{ end }}
{{ end }}

{{- .Date.Format "2006-01-02" -}}       # 获取时间。并且转换为“2006-01-02”这种格式

{{ with .Params.summary }}              # 获取summary的值
    {{ . }}
{{ end }}

{{ .Content }}                          # 获取post的“正文”内容

{{ if .Page.Params.copyright }}         # 获取copyright的值。如果值为true，则夹在版权声明模块（显示在在文章末尾处）
    {{ partial "postCopyRight.html" . }}
{{ end }}

2-2. Use Site Configuration Parameters（使用网站配置参数）

网站配置参数存放在config.toml内。

再次提醒，配置文件支持yaml，toml，json等三种格式。配置文件可以存储在config.toml之外的文件之内。

不像content（page）参数，只是针对每篇post而言，config.toml里面的参数是站点级别的，是全局的。什么意思呢？就是说content参数是每篇文章各自独立的，而存放在config.toml里面的Site configutarion parameters是全部文章共享的。

例子：

baseURL = "https://codepool.top"
languageCode = "zh-cn"

比如上面的两个参数就是针对整个网站的，换句话说就是针对所有模板的，针对所有post的，是全局的。获取它们的值的办法很简单，就是利用Site Variable。如下：

{{ .Site.BaseURL }}             # 获取baseURL的值
{{ .Site.LanguageCode }}        # 获取languageCode的值

templates介绍第三部分【完】。下一篇文章开始，我们可以编写主题了。

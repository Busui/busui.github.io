在上一篇文章我们提过，不细分的话，其实Hugo可以分为两种页面：

  1. single page（包含Regular Pages）
  2. list page（包含Home pages，Section Pages，Taxonomy List Pages，Taxonomy Terms Pages）

由于HomePage的特殊性，我们单独为它做一个页面。因此，起主要作用的文件如下：

├── _default
│   ├── baseof.html
│   ├── list.html
│   └── single.html
└── index.html

1. 完善baseof.html

我们把一些每个页面都需要的内容集中到baseof.html中。这样，就不用重复地在每个页面编写这些内容了。那每个页面都需要的内容是什么呢？我们很容易想到其中的3种类型：

  1. head标签里的内容：主要作用是提供一些十分重要的页面信息。注意它与header标签是完全不同的东西；
  2. footer标签里的内容；
  3. header标签里的内容。

一般来说，我们可以直接在baseof.html里面直接完成head/footer标签内容的编写。但正如上篇文章所说，将内容模块化，会使得程序开发变得更加清晰。baseof.html下面有两句语句：

{{- partial "head.html" . -}}
{{- partial "footer.html" . -}}

这两句语句的作用就是：在Hugo运行时，head.html和footer.html的内容会被加载到baseof.html内，从而达到和直接在baseof.html编写对应标签内容一样的效果。

1-1. 编写head.html

head标签内有很多需要告知浏览器的信息。浏览器根据这些信息，作出相应的反应。现在，head.html的内容如下：

<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">
    {{/* 网站标题，如果进入某个page，对应page的标题也会被追加 */}}
    <title>{{ block "title" . }}{{ .Site.Title }} {{ with .Params.Title }} | {{ . }}{{ end }}{{ end }}</title>
    <meta name="HandheldFriendly" content="True">
    <meta name="MobileOptimized" content="320">

    <meta name="viewport" content="width=device-width,minimum-scale=1">
    {{ hugo.Generator }}
    {{/* NOTE: 在生产环境的情况下，才允许爬虫爬取网站内容。生产环境可以通过HUGO_ENV来设置：`HUGO_ENV="production"` */}}
    {{ if eq (getenv "HUGO_ENV") "production" | or (eq .Site.Params.env "production")  }}
        <META NAME="ROBOTS" CONTENT="INDEX, FOLLOW">
    {{ else }}
        <META NAME="ROBOTS" CONTENT="NOINDEX, NOFOLLOW">
    {{ end }}

    <!-- basic favicon -->
    <link rel="shortcut icon" href="{{ relURL ($.Site.BaseURL) }}img/favicon32.png">

    {{ range .Site.Params.custom_css }}
      <link rel="stylesheet" href="{{ relURL ($.Site.BaseURL) }}{{ . }}">
    {{ end }}

    {{ with .OutputFormats.Get "RSS" }}
        <link href="{{ .RelPermalink }}" rel="alternate" type="application/rss+xml" title="{{ $.Site.Title }}" />
        <link href="{{ .RelPermalink }}" rel="feed" type="application/rss+xml" title="{{ $.Site.Title }}" />
    {{ end }}

    {{/* 生产环境下才嵌入谷歌分析 */}}
    {{ if eq (getenv "HUGO_ENV") "production" | or (eq .Site.Params.env "production")  }}
        {{ template "_internal/google_analytics_async.html" . }}
    {{ end }}
    <link rel="stylesheet" href="https://stackpath.bootstrapcdn.com/bootstrap/4.3.1/css/bootstrap.min.css" integrity="sha384-ggOyR0iXCbMQv3Xipma34MD+dH/1fQ784/j6cY/iJTQUOhcWr7x9JvoRxT2MZw1T" crossorigin="anonymous">
</head>


以.Site开头的变量是Site Varialable（网站变量）。在head.html中有三个网站变量：

  1. .Site.Title;
  2. .Site.Params.custom_css;
  3. .Site.Params.env.

这意味这我们需要在config.toml里设置这些变量的值。如下：

theme = "smalltheme"
title = "MySmalltheme"

[params]
  custom_css = ["custom_css/style.css"]             # 是一个数组，可以存放多个文件名称。
  # env = "production"                              # 发布网站的时候，要设置env的值为production。现在是development阶段，不要设置。因此将之注释掉。


在head.html中，语句：

<link rel="shortcut icon" href="{{ relURL ($.Site.BaseURL) }}img/favicon32.png">

作用是从static目录下加载img静态资源展示到浏览器tab窗口处的。我们需要准备相应的img。在smalltheme/static/目录下新建一个img文件夹。然后再放一张名字为favicon32.png，尺寸是32*32的图片到该目录下。

yellow/static/img/目录下有一张名为favicon32.png，尺寸是32*32的图片。你可以直接将之复制到smalltheme/static/img/目录下

此时，如果你运行hugo服务器，将会发现.Site.Title和favicon32.png显示出来了：

注意：如果你修改了代码，但在浏览器没有看到对应的效果，请先强制刷新浏览器。因为浏览器有缓冲机制，你修改的代码可能浏览器没有加载到。windows下chrome浏览器的强制刷新快捷键是“ctrl+F5”，MacOS则是“command+shift+r”。

那下面语句的作用是什么呢？

{{ range .Site.Params.custom_css }}
    <link rel="stylesheet" href="{{ . }}">
{{ end }}

答：作用是加载static目录下的css文件。

在config.toml中，我们已经指定了css存储的路径和名称：custom_css/style.css。那么我们需要在static目录下新建一个custom_css目录，然后再在这个目录下新建一个style.css文件。如果你没弄错，现在你的static目录结构如下：

themes/smalltheme/static
➜ tree
.
├── css         # 默认存在一个css文件夹
├── custom_css
│   └── style.css
├── img
│   └── favicon32.png
└── js          # 默认存在一个js文件夹

然后我们往style.css写点东西：

.custom-container {
    height: 100vh;
}

.custom-container类的高度属性值是100vh，也就是100%的高度。接下来，我们就可以在模板里使用custom-container这个类了。如在baseof.html中：

<div id="content" class="container bg-warning custom-container">
    {{- block "main" . }}{{- end }}
</div>

接下来，开启hugo服务器，在浏览器输入正确url即可看到效果。

1-2. 编写header.html

header和footer标签里的内容，是直接展示到浏览器页面上的。

header标签，顾名思义，就是页面的“头部”，头部一般有什么呢？一个导航栏；header.html内容如下：

<div class="container px-0">
    <nav class="navbar navbar-expand-lg navbar-dark bg-dark">
        <a class="navbar-brand font-weight-bold text-warning" href="{{ .Site.BaseURL }}">{{ .Site.Params.siteBrand | default "codepool" }}</a>
        <button class="navbar-toggler" type="button" data-toggle="collapse" data-target="#navbarNavAltMarkup"
            aria-controls="navbarNavAltMarkup" aria-expanded="false" aria-label="Toggle navigation">
            <span class="navbar-toggler-icon"></span>
        </button>
        <div class="collapse navbar-collapse" id="navbarNavAltMarkup">
            <div class="navbar-nav">
                {{ $currentPage := . }}
                {{ range .Site.Menus.main }}                
                <a class="nav-item nav-link {{ if or ($currentPage.IsMenuCurrent "main" .) ($currentPage.HasMenuCurrent "main" .) }}active{{ end }}" href="{{ .URL }}" style="text-align: center;">
                    {{ .Name }}
                    {{ if or ($currentPage.IsMenuCurrent "main" .) ($currentPage.HasMenuCurrent "main" .) }}
                        <span class="sr-only">(current)</span>
                    {{ end }}
                </a>
                {{ end }}
            </div>
        </div>
    </nav>
</div>

我们需要注意3个地方：

{{ .Site.BaseURL }}

{{ .Site.Params.siteBrand }}

{{ range .Site.Menus.main }}

以.Site开头，说明我们需要在config.toml里设置这些变量，现在config.toml变成这样了：

theme = "smalltheme"
title = "MySmalltheme"
baseURL = "http://example.com"  #.Site.BaseURL


# .Site.Menus.main
[[menu.main]]                        # config your menu  # 配置菜单
  name = "首页"
  weight = 10
  identifier = "home"
  url = "/"
[[menu.main]]
  name = "分类"
  weight = 20
  identifier = "categories"
  url = "/categories/"
[[menu.main]]
  name = "标签"
  weight = 30
  identifier = "tags"
  url = "/tags/"
[[menu.main]]
  name = "归档"
  weight = 40
  identifier = "archive"
  url = "/archive/"
[[menu.main]]
  name = "关于"
  weight = 50
  identifier = "about"
  url = "/about/"

[params]
  custom_css = ["custom_css/style.css"]             # 是一个数组，可以存放多个文件名称。
  # env = "production"                              # 发布网站的时候，要设置env的值为production。现在是development阶段，不要设置。因此将之注释掉。
  siteBrand = "MyBrand"                             # .Site.Params.siteBrand

如果没有意外，你的网站应该这样了：

你是这变换浏览器窗口大小。你会发现这个导航栏也是responsible（可响应的，即可随窗口大小而变换形态）的。你可以试着点击一下其他页面，你会发现是一片空白。我们说过，归档/分类/关于/标签都是list page。等一下我们完善list.html相关内容。那么，每一页面都可以点击了。

1-3. 编写footer.html

footer可复杂可简单。这里就编写一个十分简单的footer出来，footer.html内容如下：

<footer class="container px-0 bg-dark text-light ">
    <aside class="text-center">
        <span>Copyright &copy; 2019</span>
    </aside>
</footer>

调整style.css里面的内容：

.custom-container {
    min-height: 90vh;
}

2. 美化HomePage

我们单独为HomePage做了一个模板（index.html），在之前我们已经在index.html写了一些内容。现在我们进一步美化HomePage，内容如下：

# 用不标准的话来说，我们在baseof.html用语句“{{- block "main" . }}{{- end }}”挖了一个坑。
# 现在这里用{{ define "main" }}{{ end }}来填坑。从而，index.html就继承了baseof.html的所有内容：
# 在目前，继承的内容有：bootstrap框架资源，head内容，footer，header。
{{ define "main" }}
<div class="text-center font-weight-bold" style="font-size: 1.2rem; padding-top: 30vh;">
    <ul class="list-group list-group-flush">
        <li class="list-group-item border-0 bg-warning">知识就这样</li>
        <li class="list-group-item border-0 bg-warning">你学，或不学</li>
        <li class="list-group-item border-0 bg-warning">她都在那里</li>
        <li class="list-group-item border-0 bg-warning">没人能真正毁灭知识</li>
        <li class="list-group-item border-0 bg-warning">也没人能逼你喜欢知识</li>
    </ul>
</div>
{{ end }}
